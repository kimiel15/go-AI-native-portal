Build Guide

How to Structure Your Claude Project

To get the most out of your build phase, follow these guidelines when setting up your Claude project:
  • Start with a clear system prompt that defines Claude's role and constraints for your use case
  • Use a dedicated GitHub EMU repo — name it go-ai-native-[your-team-name]
  • Document your prompt in README.md so judges can reproduce your results
  • Test on at least 10 real production cases before the Run Phase begins on June 15
